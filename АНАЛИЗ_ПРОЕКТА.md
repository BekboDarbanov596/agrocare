# Детальный анализ проекта AI Agro Care

## 1. Общее описание

**AI Agro Care** — веб-приложение для точного земледелия и агротехнологий на PHP (MVC), с интеграцией ИИ (Hugging Face), личными кабинетами для фермеров и ветеринаров, анализом фото растений, планами участков, кейсами животных и чатом между пользователями.

## 3. Точка входа и маршрутизация

### 3.1 index.php

- Включает отображение ошибок (E_ALL).
- Запускает сессию, если ещё не запущена.
- Подключает `vendor/autoload.php` и `app/Core/helpers.php`.
- Загружает переменные из `.env` через `Dotenv::createImmutable()->load()`.
- Создаёт `Application` и вызывает `$app->run()`.
- При исключении: для путей вида `/api/*` отдаёт JSON с ошибкой; иначе — HTML-страницу 500.

### 3.2 Application и Router

- **Application**: создаёт `Request`, `Router`, `Database`, подключает `routes.php` и передаёт в него роутер; `run()` вызывает `$this->router->resolve()`.
- **Router**: хранит маршруты по методу (`get`/`post`/`put`/`delete`); при совпадении пути (в т.ч. с параметрами типа `:id`) вызывает callback (контроллер + метод) или рендерит view; для API возвращает JSON.

## 5. Регистрация и вход

### 5.1 Страницы

- **GET /login** → `AuthPageController::login()` → рендер `views/login.php`.
- **GET /register** → `AuthPageController::register()` → рендер `views/register.php`.

### 5.2 Регистрация (POST /api/auth/register)

**Клиент (register.php):** форма: email, телефон, роль (фермер / агроном / ветеринар / менеджер), пароль. JS отправляет POST на `/api/auth/register` с JSON `{ email, phone, password, role }`.

**Сервер (AuthController::register):**

- Читает из тела запроса: `email`, `phone`, `password`, `role` (по умолчанию `farmer`), опционально `vet_phone`.
- **Auth::register():**
  - Валидация: нужен email или телефон; пароль не менее 6 символов.
  - Проверка уникальности email и phone в таблице `users`.
  - Хеш пароля `password_hash(..., PASSWORD_BCRYPT)`.
  - Вставка в `users` (email, phone, password_hash, role; при роли veterinarian и наличии vet_phone — notification_settings в JSON).
  - После вставки вызывается `loginById($userId)` — в сессию пишутся `user_id` и `logged_in`.
- Ответ: JSON `{ success: true, user_id }` или `{ success: false, error }`. Код 201 при успехе, 400 при ошибке.

После успеха фронт перенаправляет на `/dashboard`.

### 5.3 Вход (POST /api/auth/login)

**Клиент (login.php):** форма: email или телефон, пароль. POST на `/api/auth/login` с `{ email, password }`.

**Сервер (AuthController::login):**

- **Auth::login($emailOrPhone, $password):**
  - Поиск пользователя по `email = :identifier OR phone = :identifier`.
  - Проверка `is_active`, проверка пароля через `password_verify`.
  - При успехе — `loginById($user['id'])`, в ответе JSON с `user: { id, email, phone, role }`.
- Ответ 200 при успехе, 401 при ошибке.

После успеха редирект на `/dashboard`.

### 5.4 Дополнительные эндпоинты авторизации

- **POST /api/auth/logout** — очистка сессии, JSON `{ success: true }`.
- **GET /api/auth/me** — текущий пользователь из сессии; при отсутствии — 401.

## 7. Использование ИИ в проекте

### 7.1 Конфигурация (app/AI/AIConfig.php)

- **API:** Hugging Face Router, OpenAI-совместимый endpoint: `https://router.huggingface.co/v1/chat/completions`.
- **Модель:** `Qwen/Qwen2.5-7B-Instruct` (fallback-модели заданы константой).
- **Ключ:** из `$_ENV['HUGGINGFACE_API_KEY']` или `HF_API_KEY`, при отсутствии — чтение из файла `.env`.
- **Параметры запроса:** temperature 0.8, max_tokens 1000, top_p 0.9; таймаут 90 сек.
- **Контекст:** до 15 последних сообщений (MAX_CONTEXT_MESSAGES).

**System prompts по context_type:**

- **general** — агроном-наставник, общие вопросы, только русский, кратко.
- **plant_analysis** — фитопатологический анализ: TRIAGE (LOW/MEDIUM/CRITICAL), что видно, диагноз, осмотр, срочные меры, прогноз.
- **animal_help** — ветеринар по виду животного (КРС, овцы, козы и т.д.): TRIAGE, дифференциальный диагноз, осмотр, первая помощь.
- **plan_advice** — планирование работ по культуре и региону: оценка пригодности, календарь работ, риски, рекомендации (даты, дозировки, препараты).

### 7.2 ChatAPI (app/AI/ChatAPI.php)

- **sendMessage($userId, $message, $contextType, $contextId, $contextData, $imagePath):**
  - Сохраняет сообщение пользователя в `ai_messages`.
  - Подгружает контекст (последние N сообщений по user_id, context_type, context_id).
  - Берёт system prompt из AIConfig по context_type (и contextData для animal_help/plan_advice/plant_analysis).
  - Формирует массив messages (system + история + текущее сообщение); при наличии `$imagePath` — мультимодальный запрос (text + image_url в base64).
  - Вызов **callHuggingFaceAPI($messages):** POST JSON в Hugging Face Router; разбор ответа из `choices[0].message.content`; постобработка: удаление китайских символов, артефактов, лишних переносов; при слишком коротком ответе — заглушка.
  - Ответ AI сохраняется в `ai_messages` с role `assistant`.
- **getChatHistory** — выборка сообщений по user_id, context_type, context_id.
- **getChatSessions** — уникальные пары (context_type, context_id) с последним сообщением и «заголовком» (первое сообщение user).
- **deleteSession** — удаление всех сообщений по user_id + context_type + context_id.
- **updateMessagesContextId** — привязка сообщений без context_id к созданному плану/анализу/кейсу.

### 7.3 AIController (API для чата)

- **POST /api/ai/chat** — авторизация обязательна. Тело: `message`, `context_type`, `context_id`, `context_data`. Для plan_advice/plant_analysis/animal_help подтягиваются данные плана/анализа/кейса и передаются в contextData. Вызов `ChatAPI::sendMessage()`.
- **GET /api/ai/history** — параметры `context_type`, `context_id`, `limit`. Возврат истории; для плана/анализа фото/кейса при пустой истории подставляется первый ответ AI из соответствующей сущности (plan_data.ai_analysis, predictions.ai_response, recommendations.ai_analysis).
- **GET /api/ai/sessions** — список сессий текущего пользователя.
- **POST /api/ai/delete-session** — удаление сессии по `context_type` и `context_id`.

### 7.4 Где ещё используется ИИ

- **Анализ фото растений (PhotoAnalysisController):** после загрузки фото создаётся запись в `plant_analyses`, изображение сжимается, вызывается метод анализа с ИИ (в т.ч. через ChatAPI с контекстом plant_analysis и путём к изображению). Результат сохраняется в `plant_analyses.predictions` (JSON, в т.ч. ai_response) и отображается на странице анализа и в чате при context_type=plant_analysis и context_id=analysis_id.
- **Планы участков (FieldPlanController / Plan):** при создании/обновлении плана может вызываться ИИ для генерации плана работ; результат сохраняется в `field_plans.plan_data` (JSON, поле ai_analysis) и показывается в чате при context_type=plan_advice.
- **Кейсы животных (AnimalCaseController):** при создании кейса вызывается ИИ для рекомендаций; результат в `animal_cases.recommendations` (JSON, ai_analysis); в чате при context_type=animal_help подставляется этот анализ.

Итого: ИИ используется как чат-консультант (общий и привязанный к плану/анализу/кейсу), для разбора фото растений и для генерации планов и ветеринарных рекомендаций.

## 8. База данных (кратко по таблицам)

- **users** — id, email, phone, password_hash, role (farmer/veterinarian/admin), language, timezone, notification_settings (JSON), is_active, created_at, updated_at.
- **farms** — хозяйства: name, region, address, owner_id.
- **user_farms** — связь пользователь–хозяйство (owner/manager/worker).
- **fields** — участки: farm_id, name, area_ha, current_crop, soil_type, coordinates (JSON), notes.
- **animals** — животные: farm_id, user_id, name, species, breed, health_status и т.д.
- **animal_cases** — кейсы: farm_id, user_id, species, symptoms, triage_level, recommendations (JSON), diagnosis, treatment, status.
- **vet_profiles** — профили ветеринаров: user_id, full_name, specialization, license_number, experience_years, bio, consultation_price, rating и т.д.
- **ai_messages** — сообщения чата с ИИ: user_id, context_type, context_id, role, content, created_at.
- **user_chats** — чаты между пользователями (user1_id, user2_id, last_message_*, unread_count_*).
- **user_messages** — сообщения в user_chats.
- **plant_analyses** — анализ фото: field_id, user_id, photo_uri, crop, predictions (JSON), confidence, status.
- **field_plans** — планы: field_id, region, crop, area_hectares, start_date, plan_data (JSON), risks (JSON), status, created_by.
- **crop_cycles** — история посевов по полю и году.
- **audit_log** — логи действий.

## 11. Чат с AI (страница /chat)

- В **chat.php** в URL задаются `context_type` и `context_id` (опционально plan_id, analysis_id, case_id).
- Загружаются сессии через GET /api/ai/sessions; список отображается в сайдбаре; при выборе сессии подгружается история GET /api/ai/history?context_type=...&context_id=...
- Отправка сообщения: POST /api/ai/chat с message, context_type, context_id.
- «Новый чат» — сброс context_id (general без привязки) или создание новой привязки к плану/анализу/кейсу.
- Удаление сессии: POST /api/ai/delete-session.

Таким образом, один интерфейс чата обслуживает и общий диалог, и контекстные (план, анализ фото, кейс животного) с подстановкой сохранённого ответа ИИ в историю при первом открытии.
